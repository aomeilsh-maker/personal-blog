## 为什么写这篇

这篇文章不是教程拼贴，而是一份完整的实战复盘：
我和 AI 从 0 开始搭建个人博客，并在真实问题中不断迭代，最终把一个“能跑”的站点打磨成“可长期使用”的内容系统。

## 项目目标

一开始目标很明确：

1. 用现代技术栈搭建个人博客（好看、轻量、可维护）
2. 采用低运维方案（不想维护云服务器）
3. 把旧站文章尽可能迁移过来
4. 持续优化视觉和信息架构

最后落地路线是：**Astro + Markdown + GitHub Pages 自动部署**。

## 第一阶段：从 0 到 1（可用版本）

### 1) 初始化项目

- 使用 Astro blog 模板快速起步
- 配置 GitHub Actions 自动部署到 Pages
- 把站点标题和简介替换成自己的信息

### 2) 首次上线

初次上线后，首页可访问，但很快遇到典型问题：

- `/blog` 404
- 文章详情页 404
- 子路径部署（`/personal-blog`）下链接异常

### 3) 关键修复点

主要修复了三类路径问题：

- `astro.config.mjs` 增加 `base` 支持
- 导航链接改为带 base 的拼接方式
- 列表页文章链接改为带 base 的完整路径

这一步让我非常确定：
**静态站点部署失败，很多时候不是“服务挂了”，而是“路径策略错了”。**

## 第二阶段：内容迁移（从旧站到新站）

### 1) 直接抓旧站失败

原计划是直接从旧站 archive + 文章详情页抓正文。
但实操发现旧站的内容接口已经异常（Notion 数据不可达），文章页返回空壳。

### 2) 改走 Notion 导出链路

最终采用：

- 导出 Notion Markdown 包
- 批量匹配旧链接 ID 与导出文件
- 写入新博客的 Markdown 内容目录
- 保留原始发布时间、标题等关键字段

这条链路稳定且可复现，适合做一次性迁移。

## 第三阶段：视觉升级（从模板感到品牌感）

我对博客做了多轮视觉优化：

- 首页加入“精选文章”封面流
- 文章列表改为封面卡片 + 信息层级
- 文章详情页优化段落、表格、引用样式
- 分类卡片加入封面图、数量与近期文章
- 加入深色模式兼容与更统一的色彩系统

核心原则是：
**先保证信息可读，再追求风格表达。**

## 第四阶段：分类系统建设

新增并固定三大分类：

- 日语学习
- AI学习
- 杂文

并为每篇历史文章补全 `category` 字段，确保后续可以持续扩展筛选、推荐和专题页。

## 这次 AI 协作里最有价值的经验

### 1) 让 AI 做“执行层”，自己把握“方向层”

- 你负责目标、风格、取舍
- AI 负责搭建、修改、验证、提交

这种分工效率很高。

### 2) 每次改动都要求“可验证”

这次我几乎每一步都做了：

- 本地 build 验证
- 线上部署检查
- 发现问题立即回滚/修复

AI 写得快，但“验证闭环”不能省。

### 3) 把一次性对话变成可复用工程资产

通过结构化改造，最终沉淀了：

- 自动部署工作流
- 分类体系
- 迁移方式
- 可持续写作目录

这比“做完一个页面”更重要。

## 下一步计划

- 增加搜索与标签页
- 增加置顶文章与专题页
- 引入评论系统（如 Giscus）
- 持续迭代视觉主题（品牌化字体与配色）

---

如果你也准备做个人博客，我的建议是：
**先跑通“写作 → 发布 → 迭代”的闭环，再慢慢打磨风格。**

博客的价值不在于第一天有多惊艳，而在于你是否能稳定写下去。

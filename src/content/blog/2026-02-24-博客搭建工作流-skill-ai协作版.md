## 这个 SKILL 解决什么问题

当你想从 0 开始搭建一个可长期维护的个人博客时，很容易陷入两个坑：

1. 一上来就追求视觉，结果部署和路径还没跑通
2. 每次重来都靠记忆，没有固定流程，效率不稳定

这个 SKILL 的目标是把“搭建-迁移-优化”的全过程模板化。

---

## SKILL 名称

`blog-system-bootstrap-and-polish`

## 适用场景

- 新建个人博客
- 从旧站迁移到静态站
- 对已有博客做结构与视觉升级

## 输入（Input）

- 博客基础信息：站点名、简介、作者信息
- 部署偏好：本地优先/静态发布
- 旧内容来源：旧站链接或导出文件
- 分类方案（建议先固定 2~5 个）

## 输出（Output）

- 可在线访问的博客
- 自动部署流水线
- 已迁移的文章内容
- 分类与可视化页面
- 一份可复用的 README 和维护策略

---

## 标准流程（Workflow）

### Step 1：初始化骨架

- 创建 Astro 项目（blog 模板）
- 本地启动并确认可运行
- 完成首个 commit

### Step 2：部署闭环

- 配置 GitHub Actions
- 开启 Pages（GitHub Actions Source）
- 设置 `SITE_URL` 与 `BASE_PATH`
- 触发首轮部署并验证线上访问

### Step 3：路径策略统一（关键）

- 统一使用 `BASE_URL` 生成站内链接
- 检查：首页、列表页、详情页、导航、返回按钮
- 修复所有子路径 404

### Step 4：内容迁移

优先级：

1. 官方导出（Markdown）
2. 可用 API
3. 页面抓取（最后手段）

迁移后必须校验：

- 标题、日期、正文完整性
- frontmatter 字段一致性
- 链接可访问性

### Step 5：信息架构

- 固定分类体系
- 批量补全文章 `category/tags`
- 新增分类聚合页

### Step 6：视觉升级

- 首页：精选流 + 最新文章
- 列表：封面卡片 + 分类徽章 + 摘要
- 详情：阅读友好排版（段落、表格、引用）
- 分类页：可视化卡片而非纯链接

### Step 7：文档化

- README 改成真实项目文档（不是模板默认文案）
- 写清：本地开发、发文方式、部署方式、变量配置

## 质量检查清单（Checklist）

### 功能检查

- [ ] 首页可访问
- [ ] `/blog` 可访问
- [ ] 文章详情可访问
- [ ] 分类页可访问
- [ ] RSS 正常

### 内容检查

- [ ] 文章数量符合预期
- [ ] 迁移文章正文完整
- [ ] 分类归档正确

### 视觉检查

- [ ] 首页、列表、详情、分类页风格统一
- [ ] 移动端可读性可接受
- [ ] 深浅色下对比度可用

## 常见失败与回退策略

### 问题 1：GitHub Pages 404

**根因**：`base` 或内部链接未按子路径处理。

**修复**：

- 校验 `astro.config.mjs` 的 `base`
- 统一站内链接生成方式

### 问题 2：旧站抓不到正文

**根因**：源站数据接口异常或权限失效。

**修复**：

- 切换到 Notion/平台官方导出链路

### 问题 3：改完样式但线上不生效

**根因**：缓存命中或旧资源回退逻辑存在。

**修复**：

- 强刷缓存
- 强制替换旧资源引用

## 推荐迭代节奏

- 第 1 周：跑通部署与迁移
- 第 2 周：完善分类与视觉统一
- 第 3 周：补搜索、标签、评论等增强项

---

## 总结

这个 SKILL 的核心不是“某个框架技巧”，而是：

> 把一次成功建站过程沉淀成可重复执行的工程脚本。

当流程被固化后，下一次搭建就不再是“重来一遍”，而是“按模板升级”。
